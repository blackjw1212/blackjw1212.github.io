# BJKW Public Console

This repository publishes two retained static products on GitHub Pages:

- `/stocks/` - 股票投資觀察台
- `/weather/` - BJKW 天氣觀察台

The root `/` is only a lightweight entry page for those two tools. No additional public surfaces are retained.

## Site Layout

```text
/
├── index.html              # Two-product entry page
├── stocks/index.html       # Stock investment observation console
├── weather/index.html      # Weather console
├── bjkw_weather.html       # Legacy redirect to /weather/
├── assets/images/          # Shared favicon and app icons
├── data/stock-risk-feed.json
├── backend/                # Taiwan market data Worker
└── weather-proxy/          # CWA weather Worker proxy
```

## Stock Investment Console

The stocks console is a static HTML app that observes selected Taiwan AI supply-chain stocks under the public stock-investment category. It displays automated market status, market index quotes, closing data, 10Y yield context, TradingView chart links, and conservative system observation prices.

Data order:

1. Cloudflare Worker proxy for `/quote`, `/eod`, and `/yield10y`.
2. Direct public TWSE end-of-day feed when available.
3. Same-origin `data/stock-risk-feed.json`.
4. Last usable browser cache.

Observation prices are generated by the page from market snapshots. They are an observation baseline only, not a target price or buy/sell advice.

## Weather Console

The weather page calls the weather proxy instead of exposing a Central Weather Administration key in static HTML.

Default proxy:

```text
https://bjkw-weather-proxy.a0926043323.workers.dev
```

The page uses:

- `/api/:endpoint` for datastore requests
- `/file/:endpoint` for file API requests

## Workers

### Taiwan Market Worker

Located in `backend/`.

Public routes:

- `GET /health`
- `GET /quote?codes=2330,2317`
- `GET /quote?indices=taiex,tpex`
- `GET /eod`
- `GET /yield10y`

`FRED_API_KEY` is optional. If it is absent, the Worker uses the US Treasury fallback for 10Y yield.

### Weather Proxy

Located in `weather-proxy/`.

Required secret:

```bash
npx wrangler secret put CWA_API_KEY
```

The Worker injects the key server-side and only allows the weather endpoints used by `/weather/`.

## Checks

Run the full local test suite:

```bash
cd backend
npm test
cd ..
node scripts/check-static-site.mjs
```

`scripts/check-static-site.mjs` enforces the retained public contract and fails if old site files, root-level favicon duplicates, old route names, old helper surfaces, or weather credentials reappear.
